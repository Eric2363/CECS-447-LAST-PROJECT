# 🎯 Purpose of This Folder

This folder stores the **professional engineering report** that documents:

- Requirements and constraints  
- Design alternatives and tradeoffs  
- System architecture  
- Module testing summary  
- System integration and validation  
- Claim–Evidence–Reasoning (CER)  
- Final demonstration results  

This report represents the **formal engineering deliverable** for the project.

# 📄 Project_Report.md

This document must:

- Include the CSULB logo on the cover page  
- Follow professional formatting  
- Use numbered sections  
- Include diagrams (block diagram, call graph, data flow)  
- Reference evidence stored in `/evidence`  
- Include 3–5 CER claims  
- Include one rejected AI suggestion (with reasoning)  

The report must reflect the **actual implemented system**, not a theoretical design.
